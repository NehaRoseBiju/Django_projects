# Cloud-Based Temperature Monitoring System

This project is a complete IoT telemetry solution. It features a Python simulation scripts representing an edge IoT sensor generating temperature readings in real-time, sending them to a secure cloud database (Supabase), and displaying updates via a premium, dark-themed responsive telemetry dashboard (built using Flask, Chart.js, and glassmorphism styling).

## Features

- **Edge Simulation**: Safe simulation loop uploading telemetry data (Temperature in °C, Date, Timestamp) to a Supabase relational table every 5 seconds. Gracefully handles internet dropouts and keyboard interrupts.
- **Dynamic Telemetry Dashboard**: A premium dark-mode custom dashboard built with a modern layout, interactive elements, custom indicators, and glowing graphs.
- **Integrated Line Chart**: Real-time rendering of Temperature vs Time logs using Chart.js with responsive resizing and smooth, flicker-free data updates.
- **Historical Querying & Filters**: Custom inputs on the dashboard to filter recorded logs between start and end Date-Time points.
- **Telemetry CSV Exporting**: Instantly download comma-separated values (CSV) matching any filtered or full records directly from the database.
- **Auto-Refresh Trigger**: Live toggle to enable or disable automatic updates (polling every 5 seconds) without reloading the page.
- **Environment Isolation**: Keeps Supabase endpoints and secret service-role API keys secure in local environment files (`.env`) excluded from code repositories.

## Installation & Setup

### 1. Prerequisites
- Python 3.8 or higher.
- A free account on [Supabase](https://supabase.com).

### 2. Database Schema Setup
Create a database table named `temp_readings` in your Supabase SQL Editor:
```sql
create table temp_readings (
  id bigint generated by default as identity primary key,
  temperature float8 not null,
  date date not null,
  "Time Stamp" timestamp with time zone not null
);
```

### 3. Clone and Install Dependencies
Navigate to the project workspace and activate your virtual environment:
```bash
# Activate existing virtual environment
..\venv\Scripts\activate

# Install required packages
pip install supabase python-dotenv flask
```

### 4. Configuration
Create a `.env` file in the root subdirectory `TemperatureMonitoring/`:
```env
SUPABASE_URL=https://<your-project-id>.supabase.co
SUPABASE_KEY=<your-anon-or-service-role-key>
```
*Note: A `.env` file is already created and ignored via `.gitignore`.*

## Access & Running the System

### Step A: Start the Data Generator
Run the python edge simulator to begin uploading simulated temperatures (ranging from 20°C to 40°C) every 5 seconds:
```bash
python data_generator.py
```
*(Press Ctrl+C to terminate).*

### Step B: Launch the Flask Server
In a separate terminal tab, run the Flask dashboard web application:
```bash
python app.py
```
By default, the server runs on `http://127.0.0.1:5000/`. Open this link in your browser to view the live telemetry dashboard.
